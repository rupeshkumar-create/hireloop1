Match scoring · formula

How overall_score is calculated

Core math: MatchingEngine._assemble_score · services/matching.py

overall = Σ (w_i × dim_i) / Σ w_included # then × gates (role · domain · title · seniority) w_skills = 0.40 w_profile = 0.30 w_experience = 0.15 w_location = 0.10 w_ctc = 0.05 # missing exp/CTC dims → weights renormed

Match scoring · dimensions

Where each dim comes from

Skills (0.40)
Lexical: 0.85×coverage + 0.15×jaccard. Then semantic lift from skills embeddings (α=0.40) — embeddings only raise the lexical base.
Profile (0.30)
Title affinity + cosine(profile_embedding, jd_embedding) with calibrated lift (α=0.50).
Experience (0.15)
Only if job seniority set — in-band 1.0, under/over linear decay.
Location (0.10)
Remote+open→1.0 · same city 1.0 · country/global 0.9 · state 0.7 · else 0.2
CTC (0.05)
Overlap vs job band (INR LPA); missing data → 0.5

Gates · floors · ranking

After the weighted mean

Gates (multipliers)

Role-fit · domain_fit · generic-title penalty (×0.35) · seniority gap (1.0 → 0.4)

Quality floors

Persist ≥ 0.35 · feed ≥ 0.38. Failed persist deletes old match_scores row.

RRF is for ordering — not the weight formula RRF: score += weight / (k + rank + 1) k = 60 Feed: fuse (overall, skills) → MMR λ=0.72 · company cap 2 Aarya job_search: RRF + 0.08·overall + 0.04·skills + 0.02·exp

Takeaways
