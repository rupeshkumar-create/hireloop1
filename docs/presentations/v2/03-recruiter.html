How the mirror job is built

ensure_role_scoring_job
Find/create jobs linked by role_id, source='recruiter'. Sync title, JD text, skills from must/nice-haves, compensation, location. Inactive until published where applicable.
score_job(job_id, limit=500)
Runs identical _assemble_score against eligible candidates.
Visibility filters
share_with_recruiters = TRUE and visibility <> 'private'.
Pipeline write
Upsert role_pipeline with match_score + criterion_scores JSON (skills / experience / location / ctc). Log recruiter_searches.

Per-criterion LLM scoring is reserved (P17+) — not live today.

Scoring recap · same engine

overall_score for candidates on a role

overall = weighted dims → × gates skills 0.40 · profile 0.30 · experience 0.15 · location 0.10 · ctc 0.05 skills_lex = 0.85×coverage + 0.15×jaccard (+ embedding lift) profile = cosine(profile_emb, jd_emb) (+ lift) persist if overall ≥ ~0.35 (feed/search floors apply on candidate side)

Recruiter cards order by match_scores.overall_score / pipeline match_score — not a second formula.

Embeddings in search

What vectors participate

Candidate

candidate_embeddings: profile · skills · resume (1536-d HNSW, cosine)

Role / job

job_embeddings: jd · title · skills — built after role sync / embed workers

If embeddings are missing, lexical/constraint dims still run — semantic lift is weaker until embed jobs catch up.
